DASHBOARD SETUP

Configuring Automatic.css for Etch

Get ACSS installed, licensed, and tuned so the generated automatic.css matches the tokens this design system expects.

ACSS 4.x Etch · zero-config Self-hosted
ℹ
Two plugins, one workflow Install Etch and Automatic.css. Per the ACSS docs, ACSS works in Etch with zero configuration — no compatibility toggles needed.
1

Install & activate the plugin

In WordPress: Plugins → Add New → Upload Plugin, choose the ACSS .zip, install and activate. A new Automatic.css item appears in the admin sidebar.

Do the same for the Etch plugin.

2

Enter your license key

Open Automatic.css → License and paste the key from your account.

!
Pick the right keyUse the key literally named –Automatic.css — not the bundle key or the plan-named key (it's usually second in the list).
3

Set website width

ACSS v4 is breakpoint-free — there is no breakpoints panel. Under Layout → Website Dimensions set just two fields. Enter everything in px — ACSS converts to rem in the generated stylesheet. This system uses:

Content Width 1200px → --content-width: 75rem (max line length) Minimum Width 360px → bottom of the fluid clamp() range Gutter 16px / 80px → 1rem → 5rem (fluid)

You type px; ACSS stores rem (1200px ÷ 16 = 75rem, 16px = 1rem, 80px = 5rem). v4 derives the fluid clamp() range for the type & spacing scales from these two values — Minimum Width (360) and Content Width (1200) — so set them here and the values below resolve correctly.

4

Configure the colour palette

In the Palette panel enter the base colour (HEX) for each palette. The six brand palettes live under Palette → Main Colors, the four status colours under Palette → Semantic Colors. ACSS reads each palette's hue from its base colour and generates the shades — so for shades you only ever set L(ightness) and C(hroma); hue is inherited.

6

Spacing & section rhythm

Under Spacing set Base Spacing and the Base Scale (mobile / desktop) — ACSS generates the whole --space-* and --section-space-* ladder from those, fluidly between Minimum Width and Content Width. These power the .section--* padding utilities and all component gaps.

7

Colour scheme (light / dark)

Two complementary v4 mechanisms:

① Contextual backgrounds — ACSS resolves text, heading, link and button colours contextually from the background utility on a section (.bg--ultra-light … .bg--ultra-dark), via Color Assignments → Light / Dark Relationships. Same markup adapts either way.

② System light/dark scheme — under Color Scheme turn on Enable Color Scheme Support (the palette is then emitted with light-dark() and shades auto-invert), pick the Website Scheme (light / dark / light dark / normal), and optionally set per-colour Main/Hover dark overrides. Use .scheme--light / .scheme--dark to force a scheme on any subtree, or Force Scheme globally. (This reference build ships single-scheme light; enable the above per project for true dark mode.)

③ Optional — Additional Styling → Effects sets global transition/easing, sticky behaviour and selection styling (the text-highlight colour, which can follow the colour scheme). Safe to leave at defaults.

8

Generate & verify

Save the dashboard — ACSS (re)generates automatic.css. Open any page in Etch; sections and containers receive their data-etch-element attributes and ACSS styling applies automatically.

✓
DonePaste components.css into Etch's global custom CSS. It references the ACSS tokens you just configured — nothing else to wire up.

How this system maps onto ACSS

Four CSS sources, loaded in this exact order, mirror the live WordPress + Etch environment:

@layer etch-reset ← WordPress/Etch reset (lowest) @layer etch-defaults ← Etch defaults automatic.css ← ACSS framework (unlayered) ← overrides layers fonts.css ← font overrides (unlayered) components.css ← our BEM blocks (unlayered) ← final say

Unlayered CSS always wins over @layer-ed CSS, so ACSS and our components override the Etch defaults with no specificity hacks — exactly how it behaves on the live site.
