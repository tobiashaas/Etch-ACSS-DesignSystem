00 — FOUNDATION

A token-driven system built on Automatic.css

Brand-agnostic components authored exactly the way they live in Etch + WordPress: strict BEM, native CSS nesting, logical properties (RTL-safe) and every value a token. This guide runs parallel to the brand design system — the brand decides colours, fonts & icons; this layer decides structure & behaviour.

The cascade — load order & who wins
1
@layer etch-reset
WordPress / Etch front-end reset — box-sizing, media, lists.
2
@layer etch-defaults
Etch low-priority defaults — easy to override.
3
automatic.css (unlayered)
The real ACSS framework — tokens + utilities. Beats both layers.
overrides ↑
4
fonts.css + components.css (unlayered)
Our blocks & type — loaded last, win on equal specificity.
final say ✓

Unlayered CSS always beats @layer-ed CSS — so ACSS & components override the Etch defaults without specificity hacks, mirroring the live WordPress environment.

Everything is local — DSGVO / GDPR
Fonts self-hosted No CDN at runtime No Google Fonts call Icons → brand layer
